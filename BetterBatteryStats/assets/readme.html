BetterBatteryStatsRelease Notes

v1.10.0.0b7 2012-08-21

  • Added raw cpu states
  • Fixed stats being displayed even if there is no reference (wakelocks, process, alarms)
  • Fixed "other" values being > 100 % (caused when timestamp of reference and sample time of stats is too far apart)
  • Fixed calculation of deep-sleep / awake not always being summed up the total "since" time

v1.10.0.0b6 2012-08-20

  • Fixed double substraction of deep sleep values sometimes causing negative values

v1.10.0.0b5 2012-08-20

  • Added battery from/to values for level and voltage to dumpfile
  • "Since" value is now consistent with references
  • Removed unneeded API calls
  • New error handling expose API errors to the user

v1.10.0.0b4 2012-08-18

  • Prevent (inconsitent) stats while plugged in
  • Fixed potential cause for refs being deleted
  • Grey background for stats bars
  • Fixed missing deep sleep in b2

v1.10.0.0b3 2012-08-18

  • Added icons to stats having package info, clicking icon starts application settings
  • Workaround to SuperUser obcessing about asking for rights
  • New error handling

v1.10.0.0b2 2012-08-17

  • Fixed FC on menu open (nested menus for SDK < 14)
  • Removed Froyo specific code as BBS does not depend on Android version for references anymore
  • Removed "test" menu

v1.10.0.0b1 2012-08-17

  • Stats requiring root (alarms/network) are disabled by default
  • Added "since boot" reference
  • If default stat type has no reference fall back to 'since boot' (is prefs are set that way)
  • Make sure references are always written (even if su rights are not granted yet)
  • Simplified Preferences
  • Added proper handling of "deleted: " kernel wakelocks
  • Added a wakelock for write operations (dumpfile, references) to make sure those are never interrupted
  • Reorganized menu
